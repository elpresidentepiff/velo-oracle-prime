## Output Standards

### Minimum Requirements (Every Race)
1. Race context + type classification
2. Pace shape + scenario probabilities
3. Intent signals (if any) and why they matter
4. Bias/conditions relevance
5. Market behaviour read
6. Threats list

### Prohibited Language
- "Guaranteed", "Sure thing", "Must win"
- "Lock", "Can't lose", "Just trust"
- Any hype or performative language

### Required Language
- "Scenario likelihood", "Risk band"
- "Evidence suggests", "Conditional on"
- "Probability", "Threats include"
- "Market indicates"

## Testing Protocol

### Shadow Races
- **Purpose:** Controlled testing ground
- **Data:** Archived races (past 12 months)
- **Goal:** Stability and accuracy validation
- **Criteria:** Output completeness, prediction accuracy

### Production Rollout
- **Step 1:** Parallel run (shadow + live)
- **Step 2:** Limited live (small subset)
- **Step 3:** Full production (all races)

## Safety Protocols

### No Guess Sentinel
- **Evidence requirement:** Every claim needs evidence
- **Confidence thresholds:** Minimum 0.5 for predictions
- **Multiple sources:** Critical claims need 2+ sources
- **Threat identification:** Every analysis must include threats

### Fail Safes
- **Output structure:** All 6 elements required
- **Language guardrail:** No prohibited language
- **Confidence guardrail:** Below 0.5 = Do not use
- **Evidence guardrail:** No evidence = No claim

## Learning Loop

### Post-Race Analysis
1. Expected vs actual
2. Assumption failures
3. Pattern tagging
4. Principle extraction
5. Memory storage

### Continuous Improvement
- **Learning velocity:** Patterns stored / time period
- **Confidence building:** Successful predictions / total
- **Pattern refinement:** New evidence + old pattern
